"# Installation Guide for AI Interview Simulator\n\nThis guide will walk you through setting up the AI-powered live coding interview simulator.\n\n## Prerequisites\n\n1. **Node.js** (v18 or higher)\n   ```bash\n   node --version\n   ```\n   Download from: https://nodejs.org/\n\n2. **npm** (comes with Node.js)\n   ```bash\n   npm --version\n   ```\n\n3. **Google Cloud Platform Account**\n   - Sign up at: https://cloud.google.com/\n   - Create a new project or use existing one\n\n4. **Git** (optional, for cloning)\n   ```bash\n   git --version\n   ```\n\n## Step 1: Google Cloud Platform Setup\n\n### 1.1 Create a GCP Project\n\n1. Go to [GCP Console](https://console.cloud.google.com/)\n2. Click \"Select a project\" → \"New Project\"\n3. Name it (e.g., \"ai-interview-simulator\")\n4. Click \"Create\"\n\n### 1.2 Enable Required APIs\n\n1. In your project, go to \"APIs & Services\" → \"Library\"\n2. Search for and enable:\n   - **Vertex AI API**\n   - **Cloud Resource Manager API**\n\n### 1.3 Create Service Account\n\n1. Go to \"IAM & Admin\" → \"Service Accounts\"\n2. Click \"Create Service Account\"\n3. Fill in:\n   - **Service account name**: `ai-interview-bot`\n   - **Description**: `Service account for AI Interview Simulator`\n4. Click \"Create and Continue\"\n5. Grant roles:\n   - Click \"Select a role\" → Search for \"Vertex AI\"\n   - Add: **Vertex AI User** and **Vertex AI Editor**\n6. Click \"Continue\" then \"Done\"\n\n### 1.4 Generate Service Account Key\n\n1. Find your service account in the list\n2. Click on it\n3. Go to \"Keys\" tab\n4. Click \"Add Key\" → \"Create new key\"\n5. Select **JSON** format\n6. Click \"Create\"\n7. **Save this JSON file** - you'll need it!\n\n### 1.5 Set Environment Variable for Service Account\n\n```bash\n# macOS/Linux\nexport GOOGLE_APPLICATION_CREDENTIALS=\"/path/to/your/service-account-key.json\"\n\n# Windows (PowerShell)\n$env:GOOGLE_APPLICATION_CREDENTIALS=\"C:\\path\\to\\your\\service-account-key.json\"\n\n# Windows (Command Prompt)\nset GOOGLE_APPLICATION_CREDENTIALS=C:\\path\\to\\your\\service-account-key.json\n```\n\n## Step 2: Install Dependencies\n\n### 2.1 Backend Dependencies\n\n```bash\ncd ai-interview-simulator\nnpm install\n```\n\n### 2.2 Frontend Dependencies\n\n```bash\ncd frontend\nnpm install\n```\n\n## Step 3: Configure Environment Variables\n\n### 3.1 Create .env File\n\n```bash\ncd ai-interview-simulator\ncp .env.example .env\n```\n\n### 3.2 Edit .env File\n\nOpen `.env` and update the following:\n\n```env\n# Google Cloud Platform Configuration\nGCP_PROJECT_ID=your-project-id  # Replace with your actual project ID\nGCP_LOCATION=us-central1        # Or your preferred region\n\n# Vertex AI / Gemini Configuration\nGEMINI_MODEL_NAME=gemini-1.5-flash\nGEMINI_SYSTEM_PROMPT=...  # Leave the default or customize
```\n\n### 3.3 Verify Project ID\n\nTo find your project ID:\n```bash\ngcloud projects list\n```\n\nOr check in GCP Console → Project Settings\n\n## Step 4: Start the Application\n\n### 4.1 Start Backend and Frontend (Development)\n\n```bash\n# From the root directory\nnpm run dev\n```\n\nThis will start:\n- **Backend API**: http://localhost:3001\n- **WebSocket**: ws://localhost:3002  \n- **Frontend**: http://localhost:5173\n\n### 4.2 Verify It's Working\n\n1. Open http://localhost:5173 in your browser\n2. You should see the landing page\n3. Click \"Start Interview\" to create a session\n\n## Step 5: Test the System\n\n### 5.1 Start an Interview\n\n1. Go to http://localhost:5173\n2. Enter your name (optional)\n3. Click \"Start Interview\"\n\n### 5.2 Use the Features\n\n- **Start Recording**: Click \"Start Recording\" to enable speech recognition\n- **Write Code**: Type in the code editor\n- **Speak**: Say something (e.g., \"I'm thinking about using a hash map...\")\n- **Request Hint**: Click \"Request Hint\" to get AI feedback\n- **End Interview**: Click \"End Interview\" when done\n\n### 5.3 Check Health Endpoints\n\n```bash\n# Check backend health\ncurl http://localhost:3001/health\n\n# Check WebSocket info\ncurl http://localhost:3001/api/ws-info\n```\n\n## Troubleshooting\n\n### Issue: \"Failed to get access token\"\n\n**Solution**: Make sure your `GOOGLE_APPLICATION_CREDENTIALS` is set correctly:\n\n```bash\necho $GOOGLE_APPLICATION_CREDENTIALS  # Should show path to JSON file\n```\n\n### Issue: \"Vertex AI request failed: 403\"\n\n**Solution**: Check service account has proper permissions:\n\n1. Go to IAM & Admin → Service Accounts\n2. Click on your service account\n3. Verify it has \"Vertex AI User\" and \"Vertex AI Editor\" roles\n\n### Issue: Port already in use\n\n**Solution**: Change ports in `.env`:\n\n```env\nPORT=3005\nWS_PORT=3006\n```\n\n### Issue: Frontend can't connect to backend\n\n**Solution**: Check CORS settings:\n\n```env\nALLOWED_ORIGINS=http://localhost:5173\n```\n\n### Issue: Speech recognition not working\n\n**Solution**: Make sure you're using a supported browser:\n- Chrome/Edge (recommended)\n- Safari\n- Firefox\n\n## Production Deployment\n\n### Build for Production\n\n```bash\nnpm run build\n```\n\n### Start Production Server\n\n```bash\nnpm start\n```\n\n### Deploy to GCP\n\nSee DEPLOYMENT.md for detailed deployment instructions.\n\n## Next Steps\n\n1. **Customize the AI**: Edit `GEMINI_SYSTEM_PROMPT` in `.env`\n2. **Add Coding Questions**: See how to add questions in the codebase\n3. **Configure Recording**: Set up screen capture for code analysis\n4. **Deploy to Production**: Follow DEPLOYMENT.md\n\n## Support\n\nIf you encounter issues:\n\n1. Check the console logs for error messages\n2. Verify all environment variables are set\n3. Check GCP billing is enabled\n4. Review Vertex AI API documentation\n\n## Quick Reference\n\n| Command | Description |\n|---------|-------------|\n| `npm run dev` | Start development servers |\n| `npm run build` | Build for production |\n| `npm start` | Start production server |\n| `npm run lint` | Check code style |\n| `npm test` | Run tests |\n\n---\n\nThat's it! You should now have a working AI interview simulator. 🚀\n"
